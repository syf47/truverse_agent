### 2_jd_shopId_comments_京东商品评_2more.md 2_jd_shopId_comments_京东商品评_2more.md
This is a database table reference document that defines the structure and attribute rules of two JD e-commerce business data tables for data development and analysis personnel. It covers two core tables: the jd_{shopId}_comments product review table and the jd_{shopId}_qa user Q&A table, both using the MergeTree engine. The review table includes 4 modules: core fields, image-related fields, interaction data, and user information, with a 365-day TTL, partitioned by review creation year-month and sorted by sku_id, create_time, and comment_id. The Q&A table stores one answer per row with redundant question information, partitioned by question creation year-month and sorted by item_id, question_id, and answer_id. Key information includes primary keys, field type descriptions, partitioning and sorting rules, and associated logical descriptions between table fields and commodity dimension tables. It is intended for big data development, data analysis, and operation personnel who need to query or process JD commodity public opinion data, requiring basic knowledge of ClickHouse table structure and e-commerce business terminology. Relevant keywords: JD product review, user Q&A, MergeTree, data table structure, sku_id.

### 京东电商_ClickHouse_数据库表结构_2more.md 京东电商_ClickHouse_数据库表结构_2more.md
This is a database table structure reference document for JD e-commerce's ClickHouse system, designed to clarify the composition and association logic of relevant business data tables. The core content is the description of table association relationships, covering three business tables under the corresponding shop ID: product table jd_{shopId}_wares, comment table jd_{shopId}_comments, and Q&A table jd_{shopId}_qa. Key information includes that the number segment 1000000266 in the table name is a sample shop ID that needs to be replaced according to actual usage; the product table is associated with the comment table through wareId and sku_id, and associated with the Q&A table through string-converted wareId and item_id; the comment and Q&A tables are also associated through product ID; each row of the Q&A table stores one answer, and question information is stored redundantly with multiple rows under the same question_id representing multiple answers. It is suitable for data analysts, database operation and maintenance personnel, and business system developers who need to use or maintain the JD e-commerce ClickHouse database, requiring basic knowledge of ClickHouse table structure and SQL association queries. Relevant keywords include JD e-commerce, ClickHouse, database table structure, product table, comment table, Q&A table, table association, shop ID.
