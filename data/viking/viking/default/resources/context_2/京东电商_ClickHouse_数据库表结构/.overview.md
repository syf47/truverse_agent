## Detailed Description
### 1_jd_shopId_wares_京东商品数据表.md 1_jd_shopId_wares_京东商品数据表.md
This is a data table reference documentation for the jd_{shopId}_wares JD product data table, aiming to standardize the structure definition and field descriptions of JD merchant product data storage for data development, analysis and operation personnel. The documentation first specifies the table's basic attributes: it adopts MergeTree engine, partitions by createTime's year and month, and sets (wareId, updateTime) as the sort key. It then elaborates all fields grouped by 8 categories: core fields, price and inventory, product status, product details, logistics attributes, extended attributes, OCR results and time fields, with clear type and business description for each field, including enumeration values for status fields like wareStatus and colType. It is applicable to scenarios such as data warehouse maintenance, product data analysis and e-commerce operation data query, requiring basic knowledge of ClickHouse table structure and e-commerce product business terms. Key keywords include JD product data, MergeTree, wareId, product status, inventory price, OCR result.

### 2_jd_shopId_comments_京东商品评_2more.md 2_jd_shopId_comments_京东商品评_2more.md
This is a data table reference document that specifies the structure and configuration of two JD e-commerce data tables for business personnel and data developers who need to use Jingdong product review and Q&A data. It covers two core tables: the jd_{shopId}_comments product review table and jd_{shopId}_qa product user Q&A table, with their respective ClickHouse engine, partition, sort key and TTL configurations specified first. The review table is divided into core fields, image-related, interaction data and user information modules; the Q&A table notes that each row stores a single answer with redundant question information, and is divided into question fields and answer fields, with field types and descriptions clearly listed for all entries. It is mainly used for data query, analysis and development scenarios related to Jingdong product user feedback. Keywords: Jingdong product review, user Q&A, MergeTree, table structure, data field, sku_id, comment_id, question_id.

### 京东电商_ClickHouse_数据库表结构_2more.md 京东电商_ClickHouse_数据库表结构_2more.md
This reference document defines the ClickHouse database table structure for JD.com e-commerce store data, serving as a guide for personnel performing data query, analysis and development related to JD store business. The document mainly covers the association relationship between three core business tables: commodity table, comment table and Q&A table. Key information includes: the shop ID segment in the table name needs to be replaced according to actual usage; the commodity table is associated with the comment table through the commodity ID, and associated with the Q&A table after the commodity ID is converted to a string; the comment table and Q&A table are also associated through the commodity ID; each row of the Q&A table stores one answer, and the question information is stored redundantly. It is intended for data analysts, backend developers and operation personnel who need to use JD store business data and have basic knowledge of ClickHouse table structure and SQL query. Keywords: JD e-commerce, ClickHouse, table structure, commodity table, comment table, Q&A table, table association.

### 常用查询模式_2more.md 常用查询模式_2more.md
This is a query reference document for JD store data analysis, aiming to provide ready-to-use SQL templates for business personnel to obtain store commodity-related data efficiently. It covers 7 common query scenarios: basic information of on-sale commodities, commodity score distribution, negative review content, image-included reviews, commodity Q&A, comprehensive review and Q&A analysis, and brand dimension analysis. It also specifies key usage notes, including dynamic shopId placeholder in table names, field association rules between commodity, comment and Q&A tables, data TTL, status filtering conditions and non-standard field spelling to avoid errors. It is suitable for operation and data analysts who need to query and analyze JD store commodity operation data, and requires basic SQL and relational database knowledge.
