## Detailed Description
### 1_jd_shopId_wares_京东商品数据表.md 1_jd_shopId_wares_京东商品数据表.md
This is a data table reference document for the Jingdong merchant product dataset `jd_{shopId}_wares`, used to standardize the structure description of the product data table stored with the MergeTree engine. The document is divided into table attribute description and 8 field modules: core fields, price and inventory, product status, product details, logistics attributes, extended attributes, OCR results, and time fields. The table is partitioned by the month of the warehousing time, sorted by product ID and data update time, covering all core dimensions of Jingdong products including basic attributes, operation status, specification parameters and data processing results. This document is applicable to data analysts, data development and operation personnel who need to use Jingdong store product data, and requires basic knowledge of ClickHouse table structure and data analysis. Keywords: Jingdong product data, MergeTree, product ID, product status, inventory price, OCR result, product attributes.

### 2_jd_shopId_comments_京东商品评_2more.md 2_jd_shopId_comments_京东商品评_2more.md
This is a database table reference document that defines the structure of two JD e-commerce data tables for data development, analysis and operation personnel with basic ClickHouse and data table knowledge. The document is divided into two main sections: the `jd_{shopId}_comments` product review table and the `jd_{shopId}_qa` user Q&A table, both using the MergeTree engine. The review table uses `create_time` for monthly partitioning, is sorted by `sku_id`, `create_time` and `comment_id`, has a 1-year TTL, and covers core review fields, image-related fields, interaction data and user information fields. The Q&A table uses `question_time` for monthly partitioning, is sorted by `item_id`, `question_id` and `answer_id`, stores one answer per row with redundant question information, and includes separate question and answer field descriptions. Key takeaways include the table storage rules, field type and definition specifications, and association relationships with product wares tables, supporting business scenarios such as user review analysis, Q&A operation and consumer insight research. Relevant keywords include JD product review, user Q&A, ClickHouse table structure, MergeTree, data partitioning.

### 京东电商_ClickHouse_数据库表结构_2more.md 京东电商_ClickHouse_数据库表结构_2more.md
This is a database table structure reference document intended to define and clarify the table design and association logic of JD e-commerce ClickHouse databases for relevant R&D engineers, data analysts, and operation and maintenance personnel who have basic knowledge of ClickHouse and JD e-commerce business. The main content it covers is the association relationship between three core business tables in the database. Key information includes: the numeric string in table names represents shop ID which needs to be replaced according to actual use scenarios; the three tables are the commodity table `jd_{shopId}_wares`, the comment table `jd_{shopId}_comments`, and the Q&A table `jd_{shopId}_qa`; the commodity table is associated with the comment table through `wareId` and `sku_id`, associated with the Q&A table through string-formatted `wareId` and `item_id`; the comment table is associated with the Q&A table through the commodity ID as well; each row of the Q&A table stores one answer, and question information is stored redundantly with the same `question_id` corresponding to multiple rows. This document is mainly used for business data query, table association development, data analysis and other scenarios related to JD store commodity, comment and Q&A data. Relevant keywords: JD e-commerce, ClickHouse, database table structure, shop ID, commodity table, comment table, Q&A table, table association.

### 常用查询模式_2more.md 常用查询模式_2more.md

The document first provides 7 commonly used query templates, covering core scenarios including basic information of on-sale commodities, commodity rating distribution, negative review content, image reviews, popular Q&A, comprehensive comment and Q&A statistics, and brand dimension analysis, all of which reserve placeholder parameters such as `shopId` and `wareId` for flexible adjustment. The last section lists 8 key usage notes, including table naming rules of different stores, association logic between commodity, comment and Q&A tables, data TTL of comment table, status judgment rules for on-sale commodities and approved images, and special field spelling reminders, which help avoid query errors. Users need to have basic SQL knowledge, and the guide is applicable to daily store operation data query, commodity reputation analysis and other work scenarios. Keywords: JD store data analysis, SQL query template, commodity information query, comment analysis, Q&A analysis, brand analysis.
