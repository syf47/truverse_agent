## Detailed Description
### 1_jd_shopId_wares_京东商品数据表.md 1_jd_shopId_wares_京东商品数据表.md
This is the data table reference document for the JD merchant commodity table, used to standardize the storage structure of commodity data for JD stores. The document is structured in two parts: basic table configuration, and field descriptions divided by business dimension. The basic configuration specifies that the table uses the MergeTree engine, is partitioned by the year and month of the storage time, and uses commodity ID and data update time as the sort key. The field descriptions cover 8 dimensions: core attributes, price and inventory, commodity status, commodity details, logistics attributes, extended attributes, OCR recognition results, and time fields, with clear definitions of each field's type and business meaning. Key information includes the commodity ID as the primary key, the definition of commodity status values, and the classification of different levels of category IDs, to support standardized data query, statistics and analysis. This document is intended for data analysts, data development engineers and business operation personnel who need to use JD store commodity data, and the user needs to have basic knowledge of ClickHouse table structure and common e-commerce business concepts. Relevant keywords include JD commodity data, MergeTree table, commodity status, commodity category, price inventory.

### 2_jd_shopId_comments_京东商品评_2more.md 2_jd_shopId_comments_京东商品评_2more.md
This is the database table reference document intended to define the structure of two JD.com e-commerce data tables for data developers, analysts and operation staff, requiring basic knowledge of ClickHouse table structures and data analysis. The document covers two core tables: the jd_{shopId}_comments commodity review table and the jd_{shopId}_qa user Q&A table. For the review table, it specifies MergeTree engine, 365-day TTL, partitioned by comment creation month, sorted by sku_id, creation time and comment ID, with fields covering review core attributes, image information, interaction data and user information. For the Q&A table, it specifies MergeTree engine, partitioned by question creation month, sorted by product ID, question ID and answer ID, storing one answer per row with redundant question information, including question and answer related fields. The document supports business scenarios such as user review analysis, commodity reputation evaluation and user demand mining.

### 京东电商_ClickHouse_数据库表结构_2more.md 京东电商_ClickHouse_数据库表结构_2more.md
This is the reference document that defines the ClickHouse database table structure for JD e-commerce store data, with the core purpose of standardizing table design and association rules for store business data storage and query. The document mainly covers the table naming rule explanation and core table association relationship specification. Key takeaways include: table names carry shopId which needs to be replaced according to actual usage; three core tables are involved, namely the product table jd_{shopId}_wares, comment table jd_{shopId}_comments, and Q&A table jd_{shopId}_qa. The product table is associated with the comment table via wareId and sku_id, associated with the Q&A table via string-formatted wareId and item_id; the comment and Q&A tables are associated via product ID, and each row of the Q&A table stores one answer with duplicate question information for the same question_id. This document is intended for data development engineers, data analysts and operation and maintenance personnel who develop or maintain JD e-commerce store data systems, requiring basic knowledge of ClickHouse table structure and SQL association queries. Relevant keywords include JD e-commerce, ClickHouse, table structure, product table, comment table, Q&A table, table association.

It is mainly used for quickly writing correct SQL statements in daily store product operation data query and analysis work.
